该文件夹下主要放置的是需要使用Git托管的代码内容
以下是改仓库内文件保管的几点注意事项，以及Git的介绍：

在所有的版本控制系统中，只能够追踪文本文件的改动
比如TXT文件，网页之类的以及所有的程序代码之类的

版本控制系统可以告诉你每次的改动，比如在第五行加了
一个单词“Linux”在第8行删除了一个单词“Windows” 而
图片、是视频这些二进制文件，虽然也能够通过版本控制
系统管理但是没办法跟踪文件的变化，只能把二进制文件
每次改动串起来，依旧是只知道图片从100kB改成了120KB，
单到底改动了什么，版本控制系统也不知道，也没有办法知道

同时Microsoft的word格式是二进制格式，因此，版本控制系
统是没办法跟踪word文件的改动的，如果要真正使用版本控制
系统，就要以纯文本的方式来编写文件。

这是因为文本是有编码的，比如中文有常用的GBK编码，日文有
Shift_JIS编码，如果没有历史遗留问题，强烈建议使用标准的
UTF-8编码，所有语言使用同一种编码，既没有冲突，又被所有
平台所支持。

值得注意的是千万不要使用Windows自带的记事本编辑任何文本文件
原因在于Microsoft开发记事本的团队使用了一个非常弱智的行为
来保存UTF-8编码的文件，他们自作聪明的在每个文件开头添加了
0xefbbbf的字符，会导致程序编译的报错，因此建议使用Visual Studio
Code代替记事本
